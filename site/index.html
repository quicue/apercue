Mechanism

How it works

1

Define a typed graph

Resources are structs with @type, depends_on, and metadata. #Graph computes topology, ancestors, and critical path at eval time.

2

Declare constraints

#Charter declares what must exist. #ComplianceCheck validates structural rules. Both produce sh:ValidationReport automatically.

3

Export projections

Every W3C artifact — JSON-LD, SHACL, SKOS, EARL, OWL-Time — is a cue export -e expression. No transformation layer. No serialization step.

Get started

Try it

# Clone the repo
$ git clone https://github.com/quicue/apercue
$ cd apercue

# Run any example
$ cue eval ./examples/course-prereqs/ -e summary
{
  graph_valid: true
  charter_complete: true
  total_resources: 12
  missing_gates: 0
}

# Export W3C projections
$ cue export ./examples/course-prereqs/ -e gaps.shacl_report --out json
$ cue export ./examples/course-prereqs/ -e type_vocabulary.skos --out json

# Use in your own project
$ mkdir myproject && cd myproject
$ cue mod init example.com/myproject@v0
$ mkdir -p cue.mod/pkg
$ ln -s ~/apercue cue.mod/pkg/apercue.ca
